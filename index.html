breakpoints. made easy.

Sass mixin for managing breakpoints inside your project with a simple, human-readable syntax.

@include breakpoints(from mobile-small to desktop-large)
View on Github Download .zip Download .tar.gz

instalation

After importing breakpoints into your project, add this at the beginning of your main Sass file:

@import 'path/to/breakpoints/src/index';

usage

@include breakpoints(from SMALL-BREAKPOINT to LARGE-BREAKPOINT) Where SMALL-BREAKPOINT and LARGE-BREAKPOINT can be a value or one of the predefined breakpoints.

The breakpoints mixin makes use of familiar syntax from the English language, such as the prepositions from and to.

For easier use, the mixin also comes with a list of predefined breakpoints. There are three types of "devices" (mobile, tablet, desktop), each one has 3 variant sizes (small, medium, large). The breakpoints are a combination of the two (mobile-small, desktop-large, etc.).

predefined breakpoints

  • mobile-small
  • mobile-medium
  • mobile-large
  • tablet-small
  • tablet-medium
  • tablet-large
  • desktop-small
  • desktop-medium
  • desktop-large

examples

custom values

.example {
    @include breakpoints(from 50em to 1024px) {
      // your CSS here
    }
  }

between two breakpoints

.example {
    @include breakpoints(from mobile-small to mobile-large) {
      // your CSS here
    }
  }

only a minimum breakpoint

.example {
    @include breakpoints(from mobile-small) {
      // your CSS here
    }
  }

only a maximum breakpoint

.example {
    @include breakpoints(to desktop-small) {
      // your CSS here
    }
  }
